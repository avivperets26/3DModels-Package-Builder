# Security Policy

## Supported Versions

Package Builder is in the repository-foundation stage and has no supported product release yet. Security fixes currently apply to the maintained `main` branch.

## Reporting a Vulnerability

Do not post vulnerability details, credentials, private keys, sensitive assets, unredacted logs, personal data, or machine-specific private paths in a public issue, discussion, pull request, or comment.

No private vulnerability-reporting channel or private contact address is documented or verified for this repository as of 2026-07-24. PB-0011 does not change GitHub repository settings. If GitHub does not visibly offer a private reporting method for this repository, do not disclose the vulnerability publicly. You may open a minimal public issue asking the maintainer to establish a private contact method, but include no vulnerability details or sensitive data.

## Current Reporting Limitation

Until a verified private channel exists, the repository cannot safely accept confidential vulnerability details. Do not send sensitive material through an unverified account or address. The full triage, response-target, emergency-patch, and disclosure procedure remains assigned to the later supply-chain and vulnerability-procedure work.

## Scope of This Policy

This file provides only the present safe-reporting boundary. It does not claim that private vulnerability reporting, custom secret-scanning patterns, automated remediation, or a complete vulnerability-response process is enabled.
